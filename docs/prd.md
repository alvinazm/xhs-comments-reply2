## 1. 产品概述

### 1.1 产品定位

小红书评论获取系统（XHS Comment System）是一款面向内容创作者和社交媒体运营者的效率工具。核心功能是通过浏览器自动化技术获取指定小红书帖子的评论数据，并利用 AI 对评论进行智能分类、自动回复，最终支持批量回复操作。

**目标用户：**
- 小红书内容创作者
- 社交媒体运营团队
- KOL 管理机构
- 需要处理大量评论反馈的账号管理者

### 1.2 核心价值

| 核心价值 | 描述 |
|---------|------|
| **自动化采集** | 替代手动复制粘贴，一键获取目标帖子的所有评论 |
| **AI 智能分类** | 利用 MiniMax 大模型对评论进行情感/意图分类（正面、问题、中性、垃圾、攻击性等） |
| **批量回复** | 支持上传 CSV 文件批量回复，或直接对已分类评论进行回复 |
| **白名单过滤** | 支持设置用户白名单，过滤不需要处理的评论 |

### 2.2 详细功能流程

#### 2.2.1 Chrome 启动与登录

**前置条件：** 无

**流程：**
1. 用户访问首页 `http://localhost:5050/`
2. 系统自动检测 Chrome 状态
3. 用户点击「启动 Chrome」按钮
4. 系统启动 Chrome Debug 浏览器
5. 自动打开小红书网页
6. **用户手动扫码登录小红书**
7. 登录状态被浏览器复用

**状态说明：**
- `checkingChrome`: 检测中（页面加载时）
- `chromeStarted`: Chrome 已启动并登录
- `chromeError`: 启动失败显示错误信息

#### 2.2.2 评论获取

**前置条件：** Chrome 已启动并登录

**流程：**
1. 用户输入小红书笔记链接
2. 设置最大评论数（默认 20）
3. 点击「获取评论」
4. 系统解析 URL 提取笔记 ID
5. 浏览器自动化打开笔记页面
6. 滚动页面加载评论
7. 按批次获取评论数据
8. 过滤白名单用户
9. 写入 CSV 文件
10. 更新任务状态和进度

**数据字段：**
```csv
评论ID, 用户ID, 用户昵称, 评论内容, 评论时间, 点赞数, 回复数
```

#### 2.2.3 AI 智能分类

**前置条件：** 评论获取任务已完成

**流程：**
1. 用户点击「AI智能分类」按钮
2. 系统使用 MiniMax API
3. 将评论分批发送给 AI（每批 20 条）
4. AI 返回分类结果
5. 进度实时更新
6. 分类完成生成新 CSV

**AI 分类标签：**
| 标签 | 说明 |
|------|------|
| `positive` / 正面 | 表达喜爱、认可、支持 |
| `question` / 问题 | 提问、寻求帮助 |
| `neutral` / 中性 | 普通评论、无明显情感倾向 |
| `constructive` / 建设性 | 提出建议、建设性意见 |
| `spam` / 垃圾 | 广告、无意义内容 |
| `hate` / 攻击性 | 恶意攻击、负面情绪 |

**CSV 新增字段：**
```csv
分类结果, 置信度, 操作建议, 分类理由, AI生成回复
```

#### 2.2.4 批量回复

**前置条件：** AI 分类已完成

**方式 A：直接回复**
1. 用户点击「直接回复」
2. 系统读取已分类 CSV
3. 对标注为 `reply` 的评论批量发送
4. 实时显示发送进度

**方式 B：上传 CSV 回复**
1. 用户准备自定义回复 CSV
2. 点击「上传 CSV 回复」
3. 系统解析 CSV
4. 预览待发送回复数
5. 用户确认发送
6. 批量执行回复

**回复数据字段：**
```csv
评论ID, 回复内容, 发送状态
```

## 5. 数据模型

### 5.1 ExportTask（导出任务）

```python
@dataclass
class ExportTask:
    task_id: str                    # 任务唯一 ID
    url: str                        # 小红书笔记链接
    max_comments: int               # 最大评论数
    status: str                    # pending/running/completed/failed
    progress: int                  # 进度百分比
    total_fetched: int             # 已获取评论数
    file_path: str                 # CSV 文件路径
    error_message: str             # 错误信息
    created_at: datetime           # 创建时间
    updated_at: datetime           # 更新时间
    completed_at: Optional[datetime]  # 完成时间
    
    # AI 分类相关
    classification_status: str     # none/running/completed/failed
    classification_progress: int    # 分类进度
    classification_summary: dict   # 分类汇总
    classified_file_path: str       # 分类后文件路径
```

## 9. 错误处理

### 9.1 常见错误

| 错误场景 | 处理方式 |
|---------|---------|
| Chrome 未启动 | 提示用户先启动 Chrome |
| URL 无效 | 提示输入有效小红书链接 |
| API Key 未配置 | 提示在设置中配置 MiniMax API Key |
| 评论获取失败 | 显示错误信息，支持重试 |
| AI 分类失败 | 显示错误信息，可重试 |
| CSV 文件不存在 | 提示文件可能被删除 |
| 回复发送失败 | 显示失败数量，可重试 |

### 9.2 日志记录

所有关键操作都会记录到日志：
- 服务器日志: `logs/server_YYYY-MM-DD.log`
- 前端日志: `logs/frontend_YYYY-MM-DD.log`
- 获取评论日志: `logs/get_comments_YYYY-MM-DD.log`
- 回复日志: `logs/reply_YYYY-MM-DD.log`
